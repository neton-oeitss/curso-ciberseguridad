FácilCTF-2-01: Modelo OSI
10 pts

Nombra las 7 capas del modelo OSI en orden, de la más baja a la más alta.

Sepáralas con comas

💡 Pista:

  • La primera capa (Capa 1) se encarga de la transmisión física de bits a través de medios como cables o ondas de radio

  • La tercera capa maneja el transporte de datos entre aplicaciones (similar a la Capa 4 de OSI)
  • La última capa combina las funciones de las capas 5, 6 y 7 de OSI

FácilCTF-2-04: Clases de IP

¿A qué clase pertenece la dirección IP 192.168.1.1?

💡 Pista:

  • La dirección 192.168.1.1 está en el rango de direcciones privadas y pertenece a una clase específica

IntermedioCTF-2-08: Subnetting
20 pts

Dada la red 192.168.1.0/24, calcula:

  1. Número de subredes posibles con máscara /26
  2. Número de hosts por subred
  3. Rango de direcciones para la primera subred

💡 Pista:

  • La máscara /24 tiene 24 bits para la red y 8 para hosts
  • La máscara /26 tiene 26 bits para la red, por lo que:
    • Bits prestados = 26 - 24 = 2 bits
    • Número de subredes = 2^2 = 4 subredes
    • Bits para hosts = 8 - 2 = 6 bits
    • Número de hosts por subred = 2^6 - 2 = 62 hosts (restamos 2 para la dirección de red y broadcast)
  • Para el rango de la primera subred:
    • Dirección de red: 192.168.1.0
    • Dirección de broadcast: 192.168.1.63
    • Rango de hosts: 192.168.1.1 a 192.168.1.62
IntermedioCTF-2-09: Firewall con iptables
20 pts

Configura un firewall para:

  1. Bloquear todo el tráfico entrante por defecto
  2. Permitir HTTP (80) y HTTPS (443)
  3. Permitir SSH (22) solo desde 192.168.1.100

Escribe las reglas de iptables

💡 Pista:

  • Para bloquear todo el tráfico entrante por defecto:
    • iptables -P INPUT DROP - Establece la política por defecto para la cadena INPUT
  • Para permitir puertos específicos:
    • iptables -A INPUT -p tcp --dport 80 -j ACCEPT - Permite tráfico TCP en el puerto 80
    • iptables -A INPUT -p tcp --dport 443 -j ACCEPT - Permite tráfico TCP en el puerto 443
  • Para restringir por dirección IP:
    • iptables -A INPUT -p tcp --dport 22 -s 192.168.1.100 -j ACCEPT - Permite SSH solo desde 192.168.1.100
  • Nota importante: Las reglas se evalúan en orden, así que las reglas de ACEPT deben ir antes que la política DROP
IntermedioCTF-2-10: TCP vs UDP
20 pts

Explica las diferencias entre TCP y UDP en términos de:

  1. Tipo de conexión
  2. Fiabilidad
  3. Velocidad
  4. Uso típico

💡 Pista:

  • Tipo de conexión:
    • TCP es orientado a conexión (requiere establecer una conexión antes de enviar datos)
    • UDP es sin conexión (envía datos sin establecer conexión previa)
  • Fiabilidad:
    • TCP garantiza la entrega de datos (usa acuses de recibo, retransmisiones, control de flujo)
    • UDP no garantiza la entrega (no tiene mecanismo de verificación)
  • Velocidad:
    • TCP es más lento debido a la sobrecarga de establecer conexión y verificar la entrega
    • UDP es más rápido ya que no tiene esta sobrecarga
  • Uso típico:
    • TCP: Navegación web (HTTP/HTTPS), correo electrónico (SMTP), transferencia de archivos (FTP)
    • UDP: Transmisión de video en tiempo real, videoconferencia, DNS, VoIP
IntermedioCTF-2-11: Resolución DNS
20 pts

Explica el proceso de resolución DNS cuando un usuario escribe www.ejemplo.com en su navegador.

Menciona al menos 4 pasos del proceso.

💡 Pista:

  • Paso 1: El navegador verifica su caché local para ver si ya tiene la dirección IP de www.ejemplo.com
  • Paso 2: Si no está en caché, el navegador consulta al resolver DNS configurado en el sistema operativo
  • Paso 3: El resolver DNS consulta sus propios registros en caché
  • Paso 4: Si el resolver no tiene la información, consulta a los servidores DNS raíz
  • Paso 5: Los servidores raíz redirigen al resolver a los servidores TLD (Top-Level Domain) para .com
  • Paso 6: Los servidores TLD redirigen al resolver a los servidores autoritativos para ejemplo.com
  • Paso 7: Los servidores autoritativos devuelven la dirección IP de www.ejemplo.com
  • Paso 8: El resolver DNS devuelve la dirección IP al navegador y la almacena en caché
AvanzadoCTF-2-15: Escaneo con Nmap
30 pts

Usa Nmap para realizar un escaneo completo de la red 192.168.1.0/24. Escribe los comandos para:

  1. Descubrir hosts activos (ping scan)
  2. Escaneo de puertos TCP abiertos (SYN scan)
  3. Detección de sistema operativo
  4. Detección de versiones de servicios
  5. Escaneo agresivo con todos los tipos de detección

Escribe los comandos que usarías

💡 Pista:

  • Descubrir hosts activos:
    • nmap -sn 192.168.1.0/24 - Realiza un ping scan para descubrir qué hosts están activos
  • Escaneo de puertos TCP:
    • nmap -sS 192.168.1.0/24 - SYN scan (sigiloso, no completa la conexión)
    • nmap -sT 192.168.1.0/24 - TCP connect scan (completa la conexión)
  • Detección de SO:
    • nmap -O 192.168.1.0/24 - Activa la detección de sistema operativo
  • Detección de versiones:
    • nmap -sV 192.168.1.0/24 - Detecta versiones de servicios
  • Escaneo agresivo:
    • nmap -A 192.168.1.0/24 - Activa detección de OS, versiones, scripts y traceroute
  • Opciones útiles:
    • -p- - Escanear todos los puertos (1-65535)
    • -T4 - Velocidad de escaneo agresiva
    • -oN output.txt - Guardar resultados en un archivo de texto

  2. 2 formas de prevenir este ataque

💡 Pista:

  • Cómo funciona: SQL Injection ocurre cuando una aplicación concatena directamente la entrada del usuario a consultas SQL sin la validación o sanitización adecuada. Esto permite a un atacante inyectar código SQL malicioso que será ejecutado por la base de datos.
  • Ejemplo de payload para autenticación bypass:
    • En un formulario de login con consulta: SELECT * FROM usuarios WHERE nombre = '$usuario' AND contrasena = '$contrasena'
    • Payload: admin' -- o admin' OR '1'='1
    • Esto hace que la condición WHERE siempre sea verdadera, permitiendo el acceso
  • Formas de prevenir:
    • Consultas parametrizadas: Usar sentencias preparadas (prepared statements) que separan el código SQL de los datos
    • Sanitización de entrada: Validar y limpiar todas las entradas de usuario antes de usarlas en consultas
    • Principio de menor privilegio: La base de datos debe ejecutarse con el mínimo de privilegios necesarios
    • ORM: Usar un mapeador objeto-relacional que maneje automáticamente la sanitización
